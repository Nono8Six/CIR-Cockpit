### 3.3 - Catalogue produit (references fabricant)

Chaque fabricant fournit un **fichier tarif** (Excel) contenant ses references produit. Ce fichier est la source de verite pour les prix publics et les conditions d'achat CIR.

**Donnees par reference produit** :

| Champ | Type | Description | Exemple |
|-------|------|-------------|---------|
| `marque` | string | Code fabricant | "SKF" |
| `reference` | string | Reference unique fabricant | "6205-2RSH" |
| `cat_fab` | string | Categorie/famille de remise fabricant | "Z16" |
| `description` | string | Designation du produit | "Rlt rigide a billes 25x52x15 2RS" |
| `description_famille` | string | Libelle de la categorie fabricant | "Roulements rigides a billes etanches" |
| `prix_public` | decimal | Prix tarif public HT | 12.50 EUR |
| `remise_standard_fab` | decimal | Remise d'achat negociee CIR | 64.0% |

**Prix d'achat calcule** :
```
PA = Prix Public x (1 - Remise Standard Fabricant / 100)
```

**Exemple** :
```
SKF 6205-2RSH :
  Prix public    = 12.50 EUR
  Remise fab     = 64%
  PA             = 12.50 x (1 - 0.64) = 4.50 EUR
```

### 3.4 - Remise fabricant : deux modes

Les fabricants gerent leurs remises d'achat de deux manieres :

**Mode 1 : Remise par famille (CAT_FAB)**
Toutes les references d'une meme categorie fabricant partagent la meme remise.
```
SKF CAT_FAB "Z16" : remise 64% ? s'applique a toutes les references Z16
  - SKF 6205-2RSH : 12.50 EUR tarif ? PA = 4.50 EUR
  - SKF 6206-2RSH : 15.80 EUR tarif ? PA = 5.69 EUR
  - SKF 6207-2RSH : 19.20 EUR tarif ? PA = 6.91 EUR
```

**Mode 2 : Remise par reference**
Chaque reference a sa propre remise d'achat.
```
TIMKEN (remises variables par reference) :
  - TIMKEN 32205 : 35.00 EUR tarif, remise 58% ? PA = 14.70 EUR
  - TIMKEN 32206 : 42.00 EUR tarif, remise 55% ? PA = 18.90 EUR
  - TIMKEN 32210 : 68.00 EUR tarif, remise 52% ? PA = 32.64 EUR
```

L'outil doit gerer les deux modes de maniere transparente.

## 4. Moteur de calcul des prix

### 4.1 - Cascade de priorite

Le prix net client est determine par une cascade de **10 niveaux de priorite**, du plus specifique au plus generique. Le moteur descend la cascade et **s'arrete au premier niveau qui produit un resultat**.

| Prio | Code | Intitule | Portee | Type de valeur |
|------|------|---------|--------|---------------|
| **0** | `PRIX_MARCHE_REF_CLT` | Prix marche client par reference | 1 client x 1 reference produit | Prix fixe EUR |
| **1** | `PRIX_MARCHE_CLT` | Prix marche client par segment | 1 client x 1 segment | Prix fixe EUR |
| **2** | `PRIX_MARCHE_GRPT` | Prix marche groupement | 1 groupement x 1 segment | Prix fixe EUR |
| **3** | `PRIX_PROMO` | Prix promotionnel | Tous clients x 1 segment | Prix fixe EUR |
| **4** | `REMISE_CLT` | Remise client | 1 client x 1 segment | % sur tarif |
| **5** | `REMISE_GRPT` | Remise groupement | 1 groupement x 1 segment | % sur tarif |
| **6** | `REMISE_CAT_TAR` | Remise categorie tarifaire | 1 categorie x 1 segment | % sur tarif |
| **7** | `REMISE_MINI` | Remise minimum | 1 segment (globale) | % sur tarif |
| **8** | `COEF_MINI` | Coefficient minimum | 1 segment (globale) | Multiplicateur PA |
| **9** | `PRIX_TARIF` | Prix tarif | 1 segment (base) | Prix EUR |

### 4.2 - Regle speciale du prix PROMO (niveau 3)

Le prix promo est une **exception** dans la cascade : il ne s'applique que s'il est **plus favorable au client** que le prix issu des niveaux 4 a 9.

**Algorithme** :
```
prix_cascade_4_plus = resoudre_cascade(niveaux 4 a 9)
si prix_promo existe ET prix_promo < prix_cascade_4_plus :
    prix_final = prix_promo
sinon :
    prix_final = prix_cascade_4_plus
```

### 4.3 - Regle du coefficient mini (niveau 8)

Le coef mini est un **plancher absolu** qui ne peut etre viole par aucune condition :
```
Prix Vente >= PA x Coef Mini
```

Si une condition (ex: remise client 50%) produit un prix inferieur au plancher coef mini, le moteur **refuse** la condition ou **alerte** l'utilisateur.

### 4.4 - Formules de conversion

#### Marge commerciale (formule CIR)
```
Marge % = (PV - PA) / PV x 100
```

#### Prix de vente depuis une marge cible
```
PV = PA / (1 - Marge% / 100)
```

#### Remise depuis un prix de vente
```
Remise % = (1 - PV / Prix Tarif) x 100
```

#### Conversion directe Marge -> Remise
```
Remise % = (1 - (PA / (1 - Marge%/100)) / Prix Tarif) x 100
```

#### Conversion directe Remise -> Marge
```
PV = Prix Tarif x (1 - Remise%/100)
Marge % = (PV - PA) / PV x 100
       = (1 - PA / (Prix Tarif x (1 - Remise%/100))) x 100
